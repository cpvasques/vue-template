# Princípios de Arquitetura Vue 3 FSD

**Você DEVE ler `.cursor/skills/vue3-fsd-development/SKILL.md` e `docs/AGENTS.md` quando:**
- Criando ou modificando estrutura de features
- Dúvidas sobre padrões arquiteturais FSD
- Decidindo onde colocar código (app, pages, features, widgets, shared)
- Questões sobre dependências entre camadas
- Nomenclatura de componentes, composables, stores e APIs

**Você DEVE ler `.cursor/skills/vue3-fsd-development/architecture.md` quando:**
- Criando novas features ou widgets
- Decidindo se algo deve ser feature vs widget vs shared
- Questões sobre hierarquia de dependências FSD
- Organizando código dentro de uma feature
- Adicionando composables, schemas ou stores a features existentes

**Você DEVE ler `.cursor/skills/vue3-fsd-development/best-practices.md` quando:**
- Implementando composables e lógica de negócio
- Dúvidas sobre boas práticas Vue 3
- Evitando anti-patterns comuns
- Implementando validação com Zod
- Gerenciando estado (Vue Query vs Pinia)

**Você DEVE ler `.cursor/skills/vue3-fsd-development/performance.md` quando:**
- Implementando lazy loading e code splitting
- Otimizando rotas e componentes
- Configurando Vue Query para performance
- Otimizando renderização e reatividade
- Analisando bundle size

**Você DEVE ler `.cursor/skills/vue3-fsd-development/patterns.md` quando:**
- Criando novas features seguindo templates
- Implementando APIs e tipos
- Criando formulários com VeeValidate
- Implementando listagens com paginação
- Criando dialogs e modals
